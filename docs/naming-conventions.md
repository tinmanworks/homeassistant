# Naming Conventions

## Branch Names

- `feature/<domain>-<short-kebab>`
- `fix/<domain>-<short-kebab>`
- `experiment/<domain>-<short-kebab>`

Examples:
- `feature/dashboards-room-split`
- `fix/automation-night-mode`
- `experiment/dashboard-hacs-popup`

## YAML File Names

Use lowercase snake_case with domain-first semantics where practical.

Examples:
- `room_workspace.yaml`
- `admin_updates.yaml`
- `automation_presence_night.yaml`

## Docs

Use kebab-case filenames.

Example:
- `ui-vs-yaml-policy.md`

## Archive Snapshots

Prefix with date for traceability.

Example:
- `2026-03-14_hacs_dashboard.yaml`
